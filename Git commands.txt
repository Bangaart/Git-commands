git init - создать гит репозиторий в выбранной папке 

git status - показывает какие файлы еще не подключены к отслеживанию а так же какие файлы изменены и нуждаються в комите

git add <.> or <"file name"> - начать отслеживать изменения в файле 

git commit - m <"commit description"> - комитим наше изменение 

git log - показывается историю комитов (кем, когда, короткое описание)

git checkout -b <branch_name> - создание новой ветки 

git checkout <branch_name> - переход на новую ветку 

git branch - m <old_name> <new_name> - переименовать ветку

git branch - d <branch_name> - удалить ветку локально

git merge <bracnh_name> - мерчь веток, если конфликт, исправляем конфлик и делаем команды  git add "" + git commit -m ""

git remote -v - проверка удаленных репозиториев которые подключены

git remote add <NameofLocalRepositpry> URL github repository (SSH or HHTPS) - добавления удаленного репозитория 

git push <NameofLocalRepository> - пуш в удаленный репозиторий 

git push -u <NameofLocalRepository> <branch_name> - пуш в этот же репозиторий нов в дургую ветку, нужно делать не с мейна а именно с той ветки которую хочешьзапушить 

git remote rename <NameofLocalRepository> <new_name> - переименовать название локального репозитория 

git remote remove <NameofLocalRepository>  - удаление локального репозитория 

git push <NameofLocalRepository> --delete <branch_name> - удалить папку на удаленном репозитории (только нужно смотреть что бы она не была дефолтной, если стоит дефолтной поменять на другую иначе не удалиться)

Команды в Git Bash консоли
clear — Очистить консоль

Навигация
pwd — Показать текущий каталог
ls - Показать файлы в данной папке, кроме скрытых
ls -f — Показать файлы в данной папке, включая и скрытые
cd c:/ — Перейти в конкретный каталог
cd - — Вернуться назад
cd .. — Выйти на 1 уровень вверх
cd ../.. — Выйти на 2 уровня вверх

Создание каталогов
mkdir — Создать папку
cd !$ — Перейти в только что созданную папку
mkdir -p {app1,app2} — Создать сразу несколько папок
mkdir -p app/{css,js} — Создать сразу несколько вложенных папок

Создание файлов
touch index.html — Создать файл index.html
touch app/{css/main.css,js/main.js,index.html} — Создать сразу несколько файлов, никаких лишних пробелов быть не должно

Удаление файлов
touch — позволяет создавать файлы
rm test — Удалить пустую папку test
rm -r test — Удалить папку test с файлами внутри неё

Перемещение файлов
mv app1/*.* app2 — Переместить все файлы из папки app1 в папку app2
